# 版本节奏

## 节奏建议
- **小版本**（patch）：bug 修复 / 文案 / 小改动 — 双周节奏。
- **中版本**（minor）：新需求 / UI 调整 — 每月一次。
- **大版本**（major）：RN 升级 / 新架构改动 — 每年 1-2 次。

## Freeze 窗口
- RN 升级期间（2026-05 中旬 — 2026-06 中旬）：除 P0 修复外，主干禁止 merge 大功能。
- 发版前 3 天进入 code freeze，仅允许回归用 fix。
