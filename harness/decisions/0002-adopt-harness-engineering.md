# ADR-0002：引入 harness/ 工程化目录

- **状态**：Accepted
- **日期**：2026-05-20

## 背景
- 项目缺少一份给 AI 协作工具（Trae / Cursor / Copilot）使用的"上下文中枢"。
- 文档分散在 README、FURTURE.md、CLEAR_CACHE.md，且未覆盖架构、需求、测试、回归、迭代。
- 在迭代过程中（特别是 RN 升级）容易丢上下文，导致重复探查。

## 决策
1. 在仓库根新增 [harness/](.. ) 目录沉淀全套 AI 工程化资产。
2. 顶层维护 [AGENTS.md](../../AGENTS.md) 作为 AI 入口，强制读完 → 再改代码。
3. 所有改动必须按 SOP 沉淀对应文档与测试。

## 备选方案
- `.ai/` 隐藏目录：被否决，原因是 GitHub 主页可见性差。
- 文档散落根目录：被否决，可维护性差，AI 难以迅速建立上下文。

## 影响
- 仓库新增一个目录，规则清晰，迁移成本低。
- 后续每次 AI 协作都有"先读 harness 再下笔"的硬约束，避免低质量产出。
